## ūüéĮ What is this?

FIX MCP is an **AI Operations Theater** ‚ÄĒ a fully interactive trading simulator built on the FIX (Financial Information eXchange) Protocol. It gives AI agents a realistic sandbox to diagnose, fix, and recover from trading infrastructure crises.

Think of it as a **cyber range for capital markets** ‚ÄĒ instead of attacking networks, you're fixing broken FIX sessions, rescuing stuck orders, and navigating cascading infrastructure failures.

## ‚ú® Features

| Feature | Details |
|---|---|
| **ūüé¨ 13 Scenarios** | Pre-built simulations covering session failures, order routing bugs, algo malfunctions, and cascading crises |
| **ūüĒß 15 Tools** | `check_fix_sessions`, `fix_session_issue`, `list_orders`, `fix_stuck_order`, `fix_order`, `cancel_order`, and more |
| **ūüďä Live Dashboard** | Real-time order book, event stream, scoring panel, and session topology visualization |
| **‚ŹĪÔłŹ Play Day Timeline** | Full trading day simulation (T+0 through market close) with scenario-specific timing |
| **ūüŹÜ Scoring System** | Objective-based point system with pro tips for optimal playthroughs |
| **ūüźô MCP Integration** | Native Model Context Protocol support ‚ÄĒ AI agents can call tools directly |

## ūüöÄ Quick Start

### Prerequisites
- **Docker & Docker Compose** ‚ÄĒ required for all services
- **Python 3.10+** (optional, for direct backend testing)

### Run the Full Stack

```bash
# Clone & start
git clone https://github.com/henryurlo/fix-mcp-server.git && cd fix-mcp-server

# Start all services (MCP server + frontend)
docker compose up -d

# Check status
docker compose logs -f
```

### Frontend UI (standalone)

```bash
# Serve the interactive simulator directly (no Docker needed for UI only)
cd fix-console

# Option 1: via Python simple server
python3 serve.py        # ‚Üí http://localhost:8088

# Option 2: direct file
# Just open frontend.html in any browser
```

The frontend auto-connects to the backend at `localhost:8000`.

### MCP Integration with AI Agents

Create or update your `.mcp.json`:

```json
{
  "mcpServers": {
    "fix-mcp": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-remote@latest"],
      "env": {
        "MCP_URL": "http://localhost:8000/mcp"
      }
    }
  }
}
```

Agents can then call FIX Protocol tools directly:

```
‚úď check_fix_sessions  ‚Üí Diagnose broken sessions
‚úď fix_session_issue   ‚Üí Auto-resolve session problems
‚úď list_orders         ‚Üí View all orders (open, stuck, completed)
‚úď fix_stuck_order     ‚Üí Rescue stuck orders by ID
‚úď fix_order           ‚Üí Fix routing, quantity, price errors
‚úď send_order          ‚Üí Place new orders with parameters
‚úď cancel_replace      ‚Üí Cancel/replace existing orders
```

## ūüéģ How to Play

1. **Pick a scenario** from the selection grid (each has a briefing, objectives, point values)
2. **Diagnose the issue** ‚ÄĒ check FIX sessions first (most problems cascade from session failures)
3. **Use the tools** ‚ÄĒ execute diagnostics, apply fixes, verify recovery
4. **Complete objectives** ‚ÄĒ each scenario has specific goals worth points
5. **Track your score** ‚ÄĒ the sidebar panel shows completed objectives and total score

**Pro tip:** Fix broken FIX sessions first. Many stuck orders are caused by session issues. Use `check_fix_sessions` to diagnose, then `fix_session_issue` to resolve.

## ūüĒß API Reference

All endpoints at `http://localhost:8000/api`

| Endpoint | Method | Description |
|---|---|---|
| `/api/scenarios` | GET | List available scenarios (name, brief, time, difficulty) |
| `/api/status` | GET | System status + tool listing |
| `/api/tool` | POST | Execute a tool (requires `{"tool_name":"...", "arguments":{}}`) |

**‚ö†ÔłŹ API Quirks:**
- `POST /api/tool` expects `"arguments"` key (not `"args"`)
- `send_order` requires `"quantity"` (not `"qty"`)
- No standalone `cancel_order` ‚ÄĒ use `cancel_replace` with `{"action":"cancel"}`
